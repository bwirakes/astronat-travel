## 1. What's fixed

### Issue 1 — geodetic zone localisation
Commit `fix(geodetic-ui): localise geodetic zone to destination longitude`.

`GeodeticLinesSection.zoneForLongitude` previously normalised longitude
to `[-180, 180]` and scanned for a zone whose `[start, start+30)` band
contained that value. For longitudes near ±180° no band qualified, and
the function silently fell back to Aries — explaining the wrong-cluster
symptom.

Replaced with a simple east-longitude index lookup:

```
floor(((lon % 360) + 360) % 360 / 30)
```

`GEODETIC_ZONES` is already ordered Aries → Pisces eastward, so the
index *is* the zone. Same fix applied to `approxAscZone` (MC + 90°).

### Issue 2 — chart-ruler "so what?" sentence
Commit `feat(reading): add chart-ruler implication sentence to Brief`.

The mechanical chart-ruler line (still rendered) tells the reader where
the ruler moves. The new second sentence, rendered immediately below in
the same italic typography, names what the trip *does* to the topic —
the gap the PDF (p.3 Jakarta/NYC example) implies but never spells out.

### Issue 3 — removed static reference sections
Commit `refactor(reading): remove static timing + framework sections`.

Deleted `TimingDecisions.tsx` and its mountpoint. Both subsections
("When to move", "How this reading is built") rendered identical text
on every reading regardless of user/city/window. Colophon is now the
page footer.

### Issue 4 — surfaced PDF principles 2, 3, 5; honestly disclaimed 6
Commit `feat(reading): surface geodetic frame + natal-geography + rule-of-three`.

- **4a** Geodetic frame (MC + ASC sign/degree) shown in the Brief as a
  separate sub-block, copy explicitly framed as "the city's", not "your".
- **4b** New `NatalGeography` section lists every natal planet's earth
  longitude + human-region label. Planets within 5° of the destination
  longitude or its opposite get an "active here" pill.
- **4c** New `ConfidenceStrip` between Brief and Best Windows tallies
  how many independent signal families fire (angle transits, parans,
  late-degree pressure, natal angle lines, ruler displacement, world
  points, natal-planet geography). ≥3 → green pill; <3 → yellow pill
  with the literal advice "interpret loosely".
- **4d** Colophon adds one mono-line acknowledging that progressions
  and solar-arc directions aren't integrated yet (PDF principle 6).

Engine math + relocate.ts + personal-lens math were untouched, per the
instructions. `personal-lens.ts` only gained the new outputs needed for
4a/4b; the existing chart-ruler computation is unchanged.

## 2. Deterministic vs AI — what generates each piece of copy

The personal reading is a hybrid. Below is exactly what is computed,
what is hard-coded, and what is generated by an LLM.

### Always deterministic (no AI in the chain)

| Surface | Source |
|---|---|
| `cityFull`, `coordsLabel`, `dateRangeLabel`, `generatedLabel`, `score`, `scoreBand` | Engine output + `Date` formatting |
| **Chart-ruler line** (Brief, italic) | Pure function `renderChartRulerLine(lens, city)` in `WeatherReading.tsx` — pulls fields from the persisted personal lens. No AI. |
| **Chart-ruler implication sentence** (NEW, Brief) | Pure function `chartRulerImplication(lens, city)` in `app/lib/personal-lens-text.ts`. Two small per-house tables: `HOUSE_NATAL_NOUN` + `HOUSE_RELOCATED_VERB`. No AI. |
| **Geodetic frame** (NEW, Brief sub-block) | `geodeticMCLongitude(lon)` + `geodeticASCLongitude(lon, lat)` from `app/lib/geodetic.ts`. Pure spherical-astronomy math. No AI. |
| **Confidence strip pill + family list** (NEW) | Counted from `events[]` (engine) + `personalLens` fields. Hard-coded labels per family. No AI. |
| **Ruler Journey** comparison block (natal vs relocated, ordinal house, ruler glyph, domain text) | Computed in `personal-lens.ts`; `HOUSE_DOMAIN` table maps house number → plain-language domain. No AI for the structured block. |
| **Where your planets land** (NEW NatalGeography section) | Computed in `personal-lens.ts`. Geographic label uses a static 12-band table (`GEOGRAPHIC_BANDS`). No AI. |
| **Geodetic Lens section** (zones, cities, keyword, desc, world map, MC/ASC panels) | Hard-coded in `app/geodetic/data/geodeticZones.ts`. No AI ever. |
| **Best Windows** (rank, dates, nights, score, bar) | `findTravelWindows()` over the day series. Dates and nights are deterministic; AI only labels (see below). |
| **Gantt timeline** | Engine score-per-day. No AI. |
| **Colophon** | Hard-coded copy + formatted date + city label. No AI. |

### AI-generated (Gemini, schema-validated)

These come from `writeWeatherReading()` in `lib/ai/prompts/geodetic-weather.ts`,
called from `runGeodeticWeather()`. The model only ever returns prose
fields; numbers and dates flow through the deterministic pipeline.

| Surface | What the LLM writes | Guard |
|---|---|---|
| **Title flourish** (`titleFlourish`) | One lowercase word in script next to city name — e.g. "opening", "window" | Schema `z.string()` |
| **Verdict / Hook / Drop line** (Why this place) | 1-line verdict, 2–3 sentence hook, single drop sentence | Schema enforces presence |
| **Movements** (`keyMoments[]`) | Title, driver, dates, body, impact for each named movement | Schema validates each field |
| **Travel-window labels + notes** (`travelWindows[].rank`, `travelWindows[].note`) | Names like "Best overall", "Quiet retreat" + a one-sentence rationale per window | Server **discards** any AI-supplied dates/nights/score and re-binds them to the deterministic candidate windows. AI cannot move a window. |
| **Ruler Journey "Chain" sentence** (`rulerJourneyChain`) | One italic blockquote sentence chaining city → relocated rising → ruler → relocated house → topic | Falls back to a deterministic chain if AI fails |
| **Advice** (`bestWindow`, `watchWindow`) | Two short sentences at the bottom of the brief | Schema-validated |

### AI-fallback (deterministic prose if the model errors)

If `writeWeatherReading()` throws, `interpretation` falls back to a
hard-coded structure built from REAL numbers (severeCount, candidateWindows,
top events). The page still ships meaningful copy without AI — only the
flourish/verdict/hook would be the deterministic version. This is how
the regenerate button can never produce a "blank reading".

### Server-side calculation (no AI, runs once per regenerate)

- `computeGeodeticWeather()` per-day forecast → `events[]`, `score`,
  `severity`, `breakdown`, `oobPlanets`, `phasesActive`.
- `computePersonalLens()` per reading → relocated angles via Swiss
  Ephemeris (`relocatedAngles`), chart-ruler house placements,
  `activeAngleLines`, `worldPointContacts`, **(new)** geodetic MC/ASC
  for the city, **(new)** `natalPlanetGeography`.
- `findTravelWindows()` → `candidateWindows[]` with deterministic
  dates/nights/score.
